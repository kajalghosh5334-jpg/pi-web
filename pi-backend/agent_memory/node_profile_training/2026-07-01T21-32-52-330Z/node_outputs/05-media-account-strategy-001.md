## Input

账号方向：AI 自媒体，目标是吸引技术开发者和有 AI 落地需求的企业主，商业模式是项目撮合交易。请输出一句话定位、双端受众画像、内容矩阵和季度规划。

## Rubric

1. 一句话定位同时能让技术者和企业主理解
2. 区分技术者和企业主的不同内容动机
3. 内容矩阵包含干货、案例、需求拆解、转介信任建设
4. 商业链路是内容到信任到撮合
5. 不编造客户案例或成交数据

## Node Output

## decision_summary
**一句话定位**  
打造“AI实战派”自媒体：通过真实技术拆解、需求拆解与开发者能力展示，成为连接技术开发者与AI落地需求方的信任枢纽，最终驱动项目撮合交易。

## positioning_or_strategy

### 双端受众画像
| 维度 | 技术开发者 | 企业主 / 需求方 |
|------|----------|----------------|
| 核心身份 | 在职工程师、独立开发者、AI研究员 | 中小企业主、业务负责人、产品经理 |
| 内容动机 | 学习前沿技术、积累项目经验、展示能力、获取接单机会 | 了解AI能做什么、评估落地成本与风险、寻找靠谱执行团队 |
| 对账号的期望 | 提供干货、同行交流、项目需求信息 | 提供真实可参考的解法、降低决策门槛、对接服务资源 |
| 内容偏好 | 技术深解、开源项目、工具评测、行业趋势 | 案例拆解（脱敏）、ROI分析、需求匹配逻辑、对接流程 |
| 抵触点 | 营销文、浅层科普、夸大宣传 | 技术黑话、无案例支撑的承诺、无法验证的团队能力 |

### 策略核心
- **双端驱动**：内容必须同时满足两类受众，不做单边知识输出。
- **信任构建**：所有内容的目标是建立“这个账号能帮你找到对的人/对的项目”的信任。
- **撮合漏斗**：内容 → 需求拆解 → 能力展示 → 交互（评论、问卷）→ 人工匹配 → 撮合服务。

## constraints
- ❗ **红线**：不得编造具体客户案例、客户名称、收益数据、成交金额或转化率。
- ❗ **红线**：不得发布未授权的真实项目复盘或企业信息。
- ❗ **红线**：不做泛AI热点账号（如只追GPT新闻），所有内容必须服务于“撮合”定位。
- **信息缺失风险**：初期无真实撮合数据时，不可伪造成功案例；须用“未知/unverified”标记，仅使用行业通用知识或脱敏模式。
- **法律/合规**：发布需求信息时需审核需求方真实性，避免成为虚假需求中转站。

## opportunities
- **双端信息差**：企业主找不到靠谱开发者，开发者找不到真实项目，账号正好充当桥梁。
- **内容差异化**：多数AI自媒体只面向C端流量，专注“撮合”可快速占领细分赛道。
- **已有生态**：可借助开源社区、AI hackathon、线上课程等渠道快速获取开发者种子用户。
- **商业模式自然延伸**：从免费内容到撮合服务费/会员费，路径清晰。（但未验证，标注为假设机会）

## execution_priorities
（按优先级排序，含内容矩阵与季度规划）

### P1：搭建内容基座（第 1 季度）
- **任务**：建立专业形象，明确定位，获取种子用户。
- **内容矩阵**：
  - 干货：每周1-2篇AI技术深度解析（如RAG优化、多模态落地难点）。
  - 工具评：主流AI工具/API实测，体现技术实力。
  - 趋势梳：行业报告解读，面向双端。
- **用户获取**：开发者社区分发、技术群分享；企业端通过创业者社群、行业媒体合作引流。
- **关键动作**：设计账号主页置顶内容，一句话说明撮合模式；搭建需求收集表单。

### P2：强化信任与需求匹配（第 2 季度）
- **任务**：开始展示撮合价值，促进双端互动。
- **内容矩阵**：
  - 案例剖析：虚构/脱敏的典型需求场景拆解（如“某电商公司如何用AI客服降本”），仅讨论方案逻辑，不绑定具体客户。
  - 需求拆解：分行业（零售、制造、金融等）分析AI落地的角色与步骤，吸引企业主对标。
  - 开发者访谈：邀请真实开发者聊项目经验、能力栈，展示人才库。
- **互动机制**：发起“你的AI难题”征集活动，收集真实需求；在评论区推动开发者自荐。
- **验证**：是否能成功收集到 N 条有效需求？是否有开发者主动申请对接？

### P3：深化撮合服务（第 3 季度）
- **任务**：将交互转化为实际撮合服务，确立商业模式。
- **内容矩阵**：
  - 撮合案例复现（脱敏、授权）：成功对接后回溯过程，展示方法论。
  - “项目英雄帖”系列：定期发布审核后的企业需求，透明介绍预算范围（如“5-15万”），吸引开发者投标。
  - 对接会/直播：组织线上撮合会，双端直谈。
- **商业动作**：试运行撮合服务费或会员制（开发者和企业主分级收费），测试价格弹性。

### P4：规模化与品牌化（第 4 季度）
- **任务**：复制模式，提升触达效率。
- **内容矩阵**：
  - 行业解决方案白皮书（输出标准文档，体现权威）。
  - 开发者能力认证/评级体系（增加信任度）。
  - 年度AI项目趋势报告，制造行业影响力。
- **运营**：尝试与孵化器、产业园合作获取批量需求；启动线上平台雏形，自动化部分匹配流程。

## validation_metrics
- **内容层面**：单篇技术干货平均互动率（评论/收藏比）、收藏/点赞比（开发者偏好深读）。
- **用户结构**：开发者关注比例 vs 企业主关注比例；若任一端占比低于 20%，内容策略需调整。
- **需求收集**：每月收集的有效企业需求数量（去重，不含广告）；每月开发者报名对接数量。
- **撮合成果**：每季度实际进入撮合流程的项目数（标记 unknown 即零起点，待运营后统计）；撮合成功数（需双方确认，严禁编造）。
- **商业指标**（Q3后）：服务费收入（或平台抽佣）增长率，客户复购率。
