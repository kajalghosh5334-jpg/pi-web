## 任务输入

```
基于上游素材，为一篇讲企业落地 LLM 真实坑的内容生成 5 组标题和封面文案。方向要覆盖技术者向、企业主向和混合向，避免标题党。
```

## 标准答案

> 标题应围绕资料乱、维护没人做、demo 到生产的距离、数据清洗/权限/评测/运维这些真实素材。技术者向可强调交付边界和维护付费；企业主向可强调为什么 LLM 项目不是买个模型；混合向可强调需求怎么拆才不会烂尾。不得写 30 天降本 80% 等无来源数字。

### 评分 Rubric

1. 输出 5 组标题和封面文案
2. 每组标注适用受众和点击动机
3. 至少覆盖技术者向、企业主向、混合向
4. 不新增上游没有支撑的案例、数字或结果
5. 避免夸张承诺和标题党

### 红线规则

1. 不得编造客户成功案例或收益数字
2. 不得输出标题党式绝对承诺
3. 不得偏离撮合交易定位
