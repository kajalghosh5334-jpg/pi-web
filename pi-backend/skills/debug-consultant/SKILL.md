# Debug Consultant — Claude 外援咨询协议

## 触发条件

只有同时满足以下条件时才启用本 skill：

1. 同一子任务已执行且失败 **≥3 次**
2. 每次失败的原因是 **逻辑 bug / 算法错误 / 不确定的正确性问题**（非工具调用失败、网络超时等基础设施问题）
3. Lead 已确认为疑难杂症，需要外援

## 工作流

### Step 1：问题分析（DeepSeek V4 Flash 完成）

```
输入：Lead 移交的任务包（含代码、报错、已尝试方案）
↓
输出：结构化问题描述（300 字以内）
```

必须包含：
- 目标函数/组件/模块位置
- 期望行为 vs 实际行为
- 已尝试的 3 种方案 + 每种失败原因
- 怀疑的根因

### Step 2：咨询 Claude（browser 工具链）

```
browser_navigate → https://claude.ai/chat
browser_type     → 粘贴问题到对话框
browser_click    → 点击发送
（等待回答）
browser_evaluate → 提取 Claude 回答全文
```

粘贴给 Claude 的问题模板：

```
我有以下疑难 bug，已尝试 3 次修复均失败。

## 代码位置
{文件路径:行号}

## 期望行为
{描述}

## 实际行为
{描述/报错}

## 已尝试方案
1. {方案一} → 失败原因
2. {方案二} → 失败原因
3. {方案三} → 失败原因

## 怀疑根因
{推理}

请给出具体的修复方案，包括代码修改和 rationale。
```

### Step 3：方案翻译与实施建议

将 Claude 的回答转换为可执行的修复步骤：

1. **改什么** — 具体文件和行号
2. **怎么改** — 代码修改
3. **验证方法** — 如何确认修复有效
4. **风险** — 修改是否影响其他功能

### Step 4：报告

输出格式：

```
## 诊断结论
{简要结论}

## Claude 建议方案
{Claude 的完整回答原文}

## 实施步骤
{可执行步骤}

## 风险评估
{影响范围}
```

## 约束

- 不允许修改未涉及的文件
- Claude 回答原文必须全文保留在输出中
- 如果 Claude 也无法解决，返回 Claude 的 exact 回答并标注"CLAUDE_UNSURE"
